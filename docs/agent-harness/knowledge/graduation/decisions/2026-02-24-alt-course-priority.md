# 대체 과목 vs 시험 통과 우선순위 결정 (구현 완료 — PR #280)

- 결정 일자: 2026-02-24 (PR #277 머지 시 변경 결정)
- 결정자: @goldm0ng (현행 구현자), @boyekim (변경 제안), @2Jin1031 (어프룹)

- 변경 PR: ~~미작성~~ → **PR #280 (2026-03-01 머지)** 로 구현 완료 — 로그인 시 대체 과목 우선 적용 (EC-015 해결됨. 2026-07-12 추기)

## 컨텍스트

영어·코딩 인증 충족 여부 판정 시, 다음 두 신호가 있음:

1. 외부 시스템 (대휴칼 등) 의 시험 통과 표시
2. allcll 백엔드의 대체 과목 이수 검사

현행 `CertificationChecker#applyAltCourse` 는:

```java
if (이미 시험 통과) return;  // 대체 과목 검사 skip
// 대체 과목 검사...
```

→ 시험 통과 > 대체 과목 우선순위

## 발견된 문제 (QA-133)

> 로그아웃 후 재 로그인 시 영어인증이 미인증으로 바뀝니다.
> ALLCLL 로그인 시, 대양휴머니티칼리지 기준으로, 영어인증을 갱신하는 것 같습니다.
> 다시 검사하기 -> 과목 등록 -> 영어 인증 완료 표시됨. -> 로그아웃 -> 로그인 -> 영어인증 미인증 표시 됨.

원인:
- 외부 시스템 (대휴칼) 의 인증 상태가 *불안정* — 로그인할 때마다 갱신되며 결과가 바뀜
- 대체 과목 이수는 *안정적인 신호* — DB 의 사용자 이수 과목 기록은 변하지 않음
- 현행 우선순위는 *불안정한 신호* (시험 통과) 를 우선시 → 사용자 결과 흔들림

## 옵션

1. 현행 유지 (시험 통과 > 대체 과목)
2. **반대로 변경 (대체 과목 > 시험 통과)**
3. 둘 다 검사 → 어느 하나라도 만족이면 통과 (사실상 옵션 2와 동등하나 명시적 OR 로직)

## 결정

**옵션 2 채택** (실제 코드 변경은 QA-133 에서)

## 근거

- 사용자 경험 일관성: 한 번 통과로 표시된 인증이 *외부 시스템 응답* 때문에 바뀌는 건 신뢰 저하
- 대체 과목 이수는 *영구적인 사실* — DB 에 박제됨. 외부 시스템에 의존하지 않음
- 학사 정책상 대체 과목 이수도 *정식 인증 통과* 와 동등한 효력

## 영향 (변경 시)

코드:
- `CertificationChecker#applyAltCourse` 의 early return 제거
- 대체 과목 이수 시 `certResult.passXxx()` 무조건 호출
- 다만 *대체 과목 미이수 + 시험 통과* 케이스는 현행 동작 유지 필요

연쇄 영향:
- 대체 과목 이수했는데 외부 시스템에서 통과 안 된 경우 → 응답에 통과로 표시됨 (현행은 미통과)
- 사용자 혼란 감소 효과
- 외부 시스템 미통과 시점에 사용자 검사 결과가 변할 우려 → 해결

## 미해결 (변경 전 검토 필요)

- 시험 *미통과* 인데 대체 과목 *미이수* 인 경우 → 현행 동작 (DB false 유지) 그대로
- 외부 시스템에서 *통과* → *미통과* 로 바뀐 경우 → 이미 cert result 가 true 면 어떻게 처리? (사용자 보호 vs 정확성 trade-off)
- 인증 종류별로 다른 우선순위가 필요한지 (영어 vs 코딩 vs 고전독서)
